管理系统前端

这是一个基于 Vue 3 + Element Plus 的管理系统前端项目，专为 UI 自动化测试而设计。

功能特性

  • 用户管理：用户列表查看、添加用户、编辑用户、删除用户
  • 订单管理：订单列表查看、订单详情、状态更新
  • 产品管理：产品列表查看、添加产品、编辑产品、库存管理
  • 仪表板：数据统计、最近活动时间线
  • 响应式设计：支持桌面和移动端访问
  • 模拟数据：包含丰富的测试数据

测试账号

  • 用户名：admin
  • 密码：123456

技术栈

  • 框架：Vue 3 (Composition API)
  • UI库：Element Plus
  • 状态管理：Pinia
  • 路由：Vue Router 4
  • 构建工具：Vite
  • 语言：JavaScript (ES6+)

项目结构

src/
├── components/          # 公共组件
│   └── MainLayout.vue   # 主布局组件
├── views/               # 页面组件
│   ├── Login.vue        # 登录页面
│   ├── Dashboard.vue    # 仪表板页面
│   ├── UserManagement.vue      # 用户管理页面
│   ├── OrderManagement.vue     # 订单管理页面
│   └── ProductManagement.vue   # 产品管理页面
├── router/              # 路由配置
│   └── index.js
├── store/               # 状态管理
│   └── index.js
├── api/                 # API接口 (预留)
├── assets/              # 静态资源
└── utils/               # 工具函数

快速开始

环境要求

  • Node.js >= 16
  • npm >= 8

安装依赖

npm install

启动开发服务器

npm run dev

项目将在 http://localhost:3000 启动

构建生产版本

npm run build

预览生产版本

npm run preview

测试友好特性

项目设计时充分考虑了自动化测试的需求：

1. 丰富的测试数据

  • 预置了大量模拟数据
  • 涵盖各种业务场景
  • 数据结构完整且真实

2. 完善的 data-testid 属性

  • 所有交互元素都有唯一的 data-testid
  • 便于编写稳定的自动化测试脚本

3. 组件化设计

  • 页面拆分为独立组件
  • 便于测试和维护

4. 状态管理集中化

  • 使用 Pinia 进行状态管理
  • 便于测试状态变化

开发说明

添加新页面

  1. 在 src/views/ 创建新页面组件
  2. 在 src/router/index.js 添加路由配置
  3. 在侧边栏菜单中添加导航项

添加新状态

在 src/store/index.js 中添加新的 Pinia store

添加新 API

在 src/api/ 目录下创建相应的 API 文件

注意事项

  • 这是一个纯前端演示项目，所有数据都是模拟的
  • 生产环境使用时需要对接真实的后端 API
  • 建议在测试环境中使用，避免数据污染
